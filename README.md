# Learn-CyberSecurity
I provide the detailed plan and resources for learning the cybersecurity from basci(no coding knowledge,no computer knowledge about ip address or how does a pc works) to a level where you can attend for GLOBAL exams

🚀 Cybersecurity Roadmap: Zero → Job Ready (Practical + No Fluff)

This repository is a **complete execution system** to go from:

👉 Beginner (no coding / no networking knowledge)  
➡️ To  
👉 Job-ready cybersecurity professional  

This is **not just a resource list**.  
This is a **structured learning + practice + career roadmap** based on real-world training experience.


🧭 HOW TO USE THIS ROADMAP

- Follow phase by phase (don’t skip)
- Focus on **practice > watching videos**
- Spend **1–2 hours daily minimum**
- Document everything on GitHub


🧱 PHASE 1: NETWORK & INTERNET BASICS (Days 1–7)

🎯 Goal:
Understand how the internet actually works (foundation of hacking)

📚 What to Learn:
- IP Address, DNS, HTTP/HTTPS
- TCP vs UDP
- OSI Model (very important)
- Ports & Protocols

🛠️ Practice:
- Check your IP using terminal
- Use `ping`, `tracert`, `netstat`

 ⚠️ Mistake to Avoid:
Jumping into hacking tools without understanding networking

🔐 PHASE 2: SECURITY FUNDAMENTALS (Days 8–14)

🎯 Goal:
Understand how attacks and defenses work

📚 What to Learn:
- CIA Triad (Confidentiality, Integrity, Availability)
- Types of attacks:
  - Phishing
  - Malware
  - DDoS
- Authentication vs Authorization
- Basics of Encryption & Hashing

🛠️ Practice:
- Identify phishing emails
- Understand password hashing


🐧 PHASE 3: LINUX MASTERY (Days 15–28)

🎯 Goal:
Become comfortable with Linux (mandatory for cybersecurity)

📚 What to Learn:
- Basic commands: `ls`, `cd`, `pwd`, `chmod`
- File permissions
- Processes & services
- Package installation

🛠️ Practice:
- Install Kali Linux / Ubuntu
- Navigate directories
- Create and manage files

⚠️ Reality:
If you are weak in Linux → you will struggle in cybersecurity


🐍 PHASE 4: PYTHON FOR CYBERSECURITY (Days 29–42)

🎯 Goal:
Automate tasks and build small security tools

📚 What to Learn:
- Variables, loops, functions
- File handling
- Basic scripting

🛠️ Build:
- Simple port scanner
- File automation script

💡 Why This Matters:
Python is used in:
- Automation
- Exploit development
- Recon tools


🌐 PHASE 5: TRAFFIC ANALYSIS (Days 43–56)

🎯 Goal:
Understand how data flows in a network

📚 What to Learn:
- Packet structure
- TCP handshake
- DNS queries

🛠️ Tools:
- Wireshark
- TCPdump

🛠️ Practice:
- Capture packets
- Analyze HTTP requests

🔧 PHASE 6: VERSION CONTROL (Days 57–63)

🎯 Goal:
Manage your projects like a professional

📚 What to Learn:
- Git basics
- Commits, branches
- GitHub usage

🛠️ Practice:
- Upload your projects
- Maintain documentation


📊 PHASE 7: SIEM & LOG ANALYSIS (Days 64–70)

🎯 Goal:
Understand how companies detect attacks

📚 What to Learn:
- Logs & monitoring
- SIEM basics
- ELK Stack (Elasticsearch, Logstash, Kibana)

🛠️ Practice:
- Analyze sample logs
- Understand alerts


☁️ PHASE 8: CLOUD SECURITY (Days 71–77)

🎯 Goal:
Understand modern infrastructure security

📚 What to Learn:
- Basics of AWS / Azure / GCP
- IAM (Identity & Access Management)
- Cloud misconfigurations


# ⚔️ PHASE 9: ETHICAL HACKING (Days 78–90)

🎯 Goal:
Perform real-world hacking techniques

📚 What to Learn:
- Reconnaissance
- Scanning
- Exploitation basics

🛠️ Tools:
- Nmap
- Burp Suite
- Metasploit

🛠️ Practice:
- TryHackMe
- HackTheBox
- VulnHub

---

WHAT YOU SHOULD BUILD (VERY IMPORTANT)

By the end, you should have:

- ✅ Nmap scan reports  
- ✅ Packet analysis reports  
- ✅ Basic Python tools  
- ✅ GitHub documentation  
- ✅ Mini penetration testing report  


CAREER PATHS

Choose your direction:

🔴 Red Team (Ethical Hacker / Pentester)  
🔵 Blue Team (SOC Analyst / Defender)  
🟣 Bug Bounty Hunter  

⚠️ REALITY CHECK

- You won’t become a hacker in 30 days  
- Watching videos ≠ skill  
- Tools ≠ knowledge  
- Practice is everything
